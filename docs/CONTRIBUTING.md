# Contributing

MAIS Log is a personal learning log. The writing is mine; the thinking is
mine; the mistakes are mine.

That said:

## What I welcome

**Corrections.** If I get a fact wrong, a citation wrong, or a claim wrong,
open an issue. Be specific: which post, which sentence, what the correction
is, and ideally a source.

**Pointers.** If you know of public work that brings together the three
pillars (clinical / AI-ML systems / security) in a way I might not have
seen, I want to know. Open an issue with the link and a sentence on why
it's relevant.

**Typos and broken links.** Open an issue or send a PR. Quick fixes are
appreciated.

## What I do not accept

**Unsolicited content contributions.** This is a personal log. The
publishing tracks (notes, synthesis, essays) are not open for guest posts.

**Style or tone changes.** The voice is mine, including the parts that
might read as awkward or non-idiomatic. Don't rewrite for style.

**Bulk edits via PR.** If you have many small fixes, open one issue listing
them; don't open one PR with sweeping changes.

## How to reach me

For non-public discussion, Mastodon DM is the preferred channel:
[@sha888@mastodon.social](https://mastodon.social/@sha888)

For everything else, GitHub issues on this repo.

## Code of conduct

Be kind. Be specific. Don't waste time — yours or mine.

If a discussion stops being useful, I will close it. That is not personal.
